# فكرة مشروع نظام الطلبيات الإلكتروني للمستودع

"حل رقمي سريع لتجميع، إدارة، وتجهيز الطلبيات بين الأقسام والمستودع"

##  الهدف من المشروع

إنهاء مشكلة **الطلبيات الورقية** العشوائية وغير الواضحة، واستبدالها بنظام إلكتروني بسيط يمكن لكل قسم استخدامه من خلال الهاتف، مع واجهة احترافية لموظفي المستودع لتنظيم واستقبال وتجهيز الطلبيات.

------

#  ماذا سيقدم النظام؟

## 1) **نظام إلكتروني لتقديم الطلبات من الأقسام**

بدل استعمال أوراق أو رسائل غير واضحة، يقوم كل قسم بتسجيل الدخول عبر رقم الهاتف ويستطيع:

- إنشاء طلبية جديدة بسهولة وبشكل واضح
- كتابة اسم المادة والكمية والملاحظات
- إرسال الطلب مباشرة إلى المستودع

### **الفائدة**

- يمنع الأخطاء في القراءة
- يمنع ازدواجية الطلبيات
- يسهل المتابعة بدل الأوراق التي تضيع أو تتكرر

------

## 2) **لوحة تحكم لموظفي المستودع**

المستودع يحصل على **لوحة تحكم واضحة** تحتوي على:

- جميع الطلبيات الواردة
- حالة كل طلب (قيد المراجعة – تجهيز – جاهز – تم التسليم…)
- تفاصيل كل طلب
- إمكانية طباعة أو استخراج PDF
- سجل كامل (History) لأي تغيير في حالة الطلب

### **الفائدة**

- تنظيم كامل للطلبيات
- معرفة ماذا تم وماذا لم يتم
- إثبات شفاف لكل خطوة

------

## 3) **واجهة للسائقين (الحركة)**

بعد تجهيز الطلب:

- يتم وضع الطلب في قائمة "جاهز للتوصيل"
- يستطيع السائق الدخول ومعرفة الطلبيات الجاهزة
- عند التوصيل يغيّر الحالة إلى "تم التسليم"

### **الفائدة**

- تنظيم عملية التوصيل
- تقليل التأخير
- متابعة التسليم بشكل واضح وموثق

------

## 4) **استخراج PDF لأي طلبية**

يتمكن المستودع من:

- استخراج ملف PDF لأي طلب
- أرشفة الطلبات
- الرجوع إليها في أي وقت

### **الفائدة**

بدل حفظ الأوراق، يصبح كل شيء مخزن إلكترونياً وقابل للبحث.

------

#  **ما الذي يحلّه النظام؟ (المشاكل التي ذكرها المسؤول)**

|              المشكلة الحالية |                كيف يحلّها المشروع؟ |
| ---------------------------: | --------------------------------: |
|  الطلبات تُرسل بطريقة عشوائية |     يصبح هناك منصة موحدة للطلبيات |
| أوراق إسمنت وكتابة غير واضحة |  نموذج إلكتروني واضح وسهل الإدخال |
|      الطلبات مزدوجة أو مكررة |     النظام يمنع ازدواجية المدخلات |
|         عدم معرفة حالة الطلب | لكل طلب حالة واضحة وتاريخ تغييرات |
|                 ضياع الأوراق |      النظام يخزن كل شيء إلكترونيًا |
|                لا يوجد توثيق |  كل طلب له PDF وسجل كامل للمتابعة |

------

#  كيف سيكون شكل النظام (بشكل مختصر)

- **تطبيق ويب** يعمل من الهاتف والكمبيوتر
- **صفحة دخول** لكل قسم
- **صفحة جديدة لإرسال طلبية**
- **لوحة تحكم للمستودع** لرؤية الطلبات
- **صفحة للسائقين** لإدارة التوصيل
- **قاعدة بيانات على الإنترنت** تحفظ كل شيء
- **قابل للتطوير لاحقاً** ليصبح نظام إدارة مخزون كامل

------

#  لماذا هذا النظام مفيد وسريع التنفيذ؟

- يعتمد على ويب — لا يحتاج تثبيت برامج
- يعمل من أي جهاز
- واجهات بسيطة جداً
- يمكن إطلاق النسخة الأولى خلال فترة قصيرة
- يوفر الوقت ويمنع الأخطاء، ويوثق كل العمليات

------

#  الخلاصة

النظام المقترح هو **حل سريع وفعّال** يربط الأقسام والمستودع والحركة بآلية سلسة ومنظمة، وينهي مشكلة الفوضى الحالية في الطلبات الورقية، ويقدم أساس قوي للتوسع مستقبلاً إلى نظام مخازن متكامل.
